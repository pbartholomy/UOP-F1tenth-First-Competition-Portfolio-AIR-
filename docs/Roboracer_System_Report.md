**Config path:** `~/.var/app/com.vesc_project.VescTool/config/VESC/VESC Tool.conf`

The **VESC (Vedder Electronic Speed Controller)** is the hardware that drives the brushless motor and servo steering on the F1TENTH car. VESC Tool is the graphical application used to configure it.

| `js_erpm_min` | -30,000 | Max reverse motor speed (ERPM) |

The VESC receives drive commands from the ROS2 stack (typically Ackermann steering messages) and converts them into PWM signals for the motor and servo.

## 8. Setup & Troubleshooting Notes

### `~/Final Step by Step Setup` (plain text file)

A personal step-by-step guide for running the full Isaac ROS + ZED demo stack:

```
1. cd ~/workspaces/isaac-ros-dev/src/isaac_ros_common/scripts
2. ./run_dev.sh                          ← Start the Isaac ROS Docker container
3. Inside Docker: chmod +x install-zed-aarch64.sh
4. Inside Docker: run install-zed-aarch64.sh     ← Install ZED SDK inside Docker
5. cd ~/workspaces/isaac-ros-dev/src
6. sudo apt update
7. rosdep update && rosdep install --from-paths src/zed-ros2-wrapper --ignore-src -r -y
8. colcon build --symlink-install --packages-up-to zed_wrapper   ← Build ZED ROS2 driver
9. Run ZED Explorer at /usr/local/zed/tools/ to verify camera
10. Open new terminal tab → re-enter Docker via run_dev.sh
```

### `~/Desktop/SSD Bootloader Bug Cant find Partition` (plain text file)

Troubleshooting note documenting a Jetson SSD boot failure and the fix:

- **Problem:** SSD stopped booting after a system update — the partition UUID in the boot config pointed to the wrong location
- **Fix:** 
  1. Plug SSD into another Jetson or computer
  2. Use `lsblk` and `blkid` to find the correct partition UUID
  3. Edit `/etc/fstab` with the correct UUID
  4. In the Jetson boot menu, switch the boot manager from `extlinux` to `kernel`
- The `/boot/extlinux/extlinux.conf` file was also edited but may not have been strictly necessary
